Create a playground

Playgrounds are an interactive sandbox environment that enables you to learn and explore Swift and visualize the results of your code in real-time after each edit and without requiring you to manually build and run.

Creating a playground

  1. Launch Xcode if it's not already running
    • click get started with a playground in the welcome window
    • or select file>new playground
  3. Accept the suggested file name or give it a new name, and select iOS for the platform
  4. Click create

Log to the console

Printing info to the console is a fundamental aspect of programming in any language

The Swift standard library includes the print function, which is used to print a textual representation of a value to the standard output. 


print(str)
//  Print automatically prints a new line character at the end

NSLog(str)
// NSLog is still available and a great choice if you want to include
// a time stamp in your printout.

print(hello, playground)
// Print can print multiple items passed to it in a comma separated list too.
// This kind of parameter is called a variatic parameter
// because it can take one or more values in a comma separated list.

Print offers some additional options which can be seen by option clicking on print to access it's quick help documentation in a popup. 


print(hello, playground, separator: "_", terminator: " ")
// change Prints default separator and terminator

print("...")
// Because we've specified an empty string for the terminator the
// next printed message will also begin on the same line.

Print can use string interpolation to print values or simple expressions that return a value embedded into a string. To include a value or expression is a string to be printed, write a backslash and then enclose the value or expression in parentheses. 


print("\(hello) viewer!")

print("1 + 1 = \(1 + 1)")
// print the result of an an expression

Literal expressions available in Swift that you can use to print useful information to the console. 


// creating a function that prints examples of each of these literals.
func printLiteralExpressions() {
    print("Function: \(#function)")
    print("File: \(#file)")
    print("Line: \(#line)")
    print("Column: \(#column)")
}

printLiteralExpressions()
// The result of printing these literals can be seen in the console.

Add comments

adding comments to your code can go a long way towards helping your fellow programmers understand how to work with the code you've written, so it has to provide details about parameters from method or example use cases.

The syntax to write a single-line comment is to write two forward slashes, followed by your comment text 


// Single-line comment

Single-line comments also support using landmarks, which will be displayed in the jump bar, and that runs along the top of the source editor and is currently displaying the playground file name.

*Add image here

There are three landmarks available in Swift, Mark, ToDo, and FixMe. 


// MARK: This is viewable in the jumb bar

// TODO: Do this

// FIXME: Fix this

// MARK: - Add a separator above this

these landmarks can be searched, which makes them useful for quickly being able to find all tasks or issues that need attention.

Multi-line comments can be created by writing a forward slash, an asterisk to start, and an asterisk forward slash to end. 


func someFunction() {
    /* A multi-line comment
    nested in a function
    */
}

Xcode supports using markup syntax to create rich text comments, which are referred to as delimited comments, or often just delimiters.

Apple's markup formatting reference documentation details using markup in comments.

Run a REPL

A REPL, or read eval print loop, can be initiated in a command line environment such as from the terminal app or in an Xcode project console during a paused debug session.

To start a REPL session using terminal, which can be launched by using Spotlight, but you can also find it in the applications utilities folder type swift at the command prompt and press return.

To end a REPL, type colon quit, or just colon Q.
